# Multicast Group Manager Callbacks

The multicast group manager uses the following callbacks to notify clients (typically, routing protocols) of events and state changes:

[**PMGM\_CREATION\_ALERT\_CALLBACK**](/windows/win32/Mgm/nc-mgm-pmgm_creation_alert_callback?branch=master)

[**PMGM\_JOIN\_ALERT\_CALLBACK**](/windows/win32/Mgm/nc-mgm-pmgm_join_alert_callback?branch=master)

[**PMGM\_PRUNE\_ALERT\_CALLBACK**](/windows/win32/Mgm/nc-mgm-pmgm_prune_alert_callback?branch=master)

[**PMGM\_LOCAL\_JOIN\_CALLBACK**](/windows/win32/Mgm/nc-mgm-pmgm_local_join_callback?branch=master)

[**PMGM\_LOCAL\_LEAVE\_CALLBACK**](/windows/win32/Mgm/nc-mgm-pmgm_local_leave_callback?branch=master)

[**PMGM\_RPF\_CALLBACK**](/windows/win32/Mgm/nc-mgm-pmgm_rpf_callback?branch=master)

[**PMGM\_WRONG\_IF\_CALLBACK**](/windows/win32/Mgm/nc-mgm-pmgm_wrong_if_callback?branch=master)

The multicast group manager uses the following callbacks to notify IGMP of events and state changes:

[**PMGM\_DISABLE\_IGMP\_CALLBACK**](/windows/win32/Mgm/nc-mgm-pmgm_disable_igmp_callback?branch=master)

[**PMGM\_ENABLE\_IGMP\_CALLBACK**](/windows/win32/Mgm/nc-mgm-pmgm_enable_igmp_callback?branch=master)
