# Server Data Objects Interfaces

The Server Data Objects (SDO) API exposes the following interfaces.



<table>
<colgroup>
<col style="width: 50%" />
<col style="width: 50%" />
</colgroup>
<thead>
<tr class="header">
<th>Interface</th>
<th>Purpose</th>
</tr>
</thead>
<tbody>
<tr class="odd">
<td>[<strong>ISdo</strong>](https://msdn.microsoft.com/library/bb960639)</td>
<td>Manipulate an SDO object.<br/></td>
</tr>
<tr class="even">
<td>[<strong>ISdoCollection</strong>](https://msdn.microsoft.com/library/bb960640)</td>
<td>Manipulate a collection of SDO objects.<br/></td>
</tr>
<tr class="odd">
<td>[<strong>ISdoDictionaryOld</strong>](https://msdn.microsoft.com/library/bb960649)</td>
<td>Manipulate the attribute dictionary.<br/></td>
</tr>
<tr class="even">
<td>[<strong>ISdoMachine</strong>](https://msdn.microsoft.com/library/bb960655)</td>
<td>Manage the SDO computer, and retrieve SDO objects.<br/></td>
</tr>
<tr class="odd">
<td>[<strong>ISdoServiceControl</strong>](https://msdn.microsoft.com/library/bb960664)</td>
<td>Manage the service being administered through SDO, that is, either Internet Authentication Service (IAS) or Remote Access Server (RAS).<br/>
<blockquote>
[!Note]<br />
Internet Authentication Service (IAS) was renamed Network Policy Server (NPS) starting with Windows Server 2008.
</blockquote>
<br/></td>
</tr>
</tbody>
</table>
