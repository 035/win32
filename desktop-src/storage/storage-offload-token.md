# STORAGE\_OFFLOAD\_TOKEN structure

The **STORAGE\_OFFLOAD\_TOKEN** structure contains a token value that serves as a representation of a data set range within a file on a volume. This structure is used in performing offload reads and writes.

## Syntax


```C++
typedef struct _STORAGE_OFFLOAD_TOKEN {
  UCHAR TokenType[4];
  UCHAR Reserved[2];
  UCHAR TokenIdLength[2];
  union {
    struct {
      UCHAR Reserved2[STORAGE_OFFLOAD_TOKEN_ID_LENGTH];
    } StorageOffloadZeroDataToken;
    UCHAR  Token[STORAGE_OFFLOAD_TOKEN_ID_LENGTH];
  };
} STORAGE_OFFLOAD_TOKEN, *PSTORAGE_OFFLOAD_TOKEN;
```



## Members

<dl> <dt>

**TokenType**
</dt> <dd>

The type of token contained in this structure. **TokenType** is set to either of the following:



| Value                                                                                                                                                                                                                                  | Meaning                                                                                                            |
|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------|
| <span id="STORAGE_OFFLOAD_TOKEN_TYPE_ZERO_DATA"></span><span id="storage_offload_token_type_zero_data"></span><dl> <dt>**STORAGE\_OFFLOAD\_TOKEN\_TYPE\_ZERO\_DATA**</dt> </dl> | A well-known token format is contained in **Token**. The token represents data considered as all zeros.<br/> |
| <span id="Any_other_value"></span><span id="any_other_value"></span><span id="ANY_OTHER_VALUE"></span><dl> <dt>**Any other value**</dt> </dl>                                   | A vendor-specific token format is contained in **Token**.<br/>                                               |



 

</dd> <dt>

**Reserved**
</dt> <dd>

Reserved.

</dd> <dt>

**TokenIdLength**
</dt> <dd>

The length of the token data in **Token**.

</dd> <dt>

**StorageOffloadZeroDataToken**
</dt> <dd>

The token value when the data it represents is considered as all zeros. This member is valid when **TokenType** is set to **STORAGE\_OFFLOAD\_TOKEN\_TYPE\_ZERO\_DATA**.

<dl> <dt>

**Reserved2**
</dt> <dd>

Reserved.

</dd> </dl> </dd> <dt>

**Token**
</dt> <dd>

The vendor-supplied token value representing a data set range for an offload read or write operation.

</dd> </dl>

## Remarks

The token information contained in **STORAGE\_OFFLOAD\_TOKEN** is generated by the offload read/write infrastructure. All members of this structure should be considered as read only and must not be modified by any callers requesting an offload read or an offload write.
