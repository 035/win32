# Ntdddisk.h

This section contains reference topics for the Ntdddisk.h header.

## In this section



| Topic                                                                                | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
|--------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [Disk I/O Control Codes](disk-i-o-control-codes.md)<br/>                      |                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| [**CREATE\_DISK**](create-disk.md)<br/>                                       | The CREATE\_DISK structure is used with the [**IOCTL\_DISK\_CREATE\_DISK**](ioctl-disk-create-disk.md) IOCTL to initialize a disk with an empty partition table. The partition table styles are master boot record (MBR) or GUID partition table (GPT).<br/>                                                                                                                                                                                               |
| [**CREATE\_DISK\_GPT**](create-disk-gpt.md)<br/>                              | The CREATE\_DISK\_GPT structure is used with the IOCTL [**IOCTL\_DISK\_CREATE\_DISK**](ioctl-disk-create-disk.md) to initialize a disk with an empty GPT partition table.<br/>                                                                                                                                                                                                                                                                             |
| [**CREATE\_DISK\_MBR**](create-disk-mbr.md)<br/>                              | The CREATE\_DISK\_MBR structure is used with the IOCTL [**IOCTL\_DISK\_CREATE\_DISK**](ioctl-disk-create-disk.md) to initialize a disk with an empty MBR partition table.<br/>                                                                                                                                                                                                                                                                             |
| [**DETECTION\_TYPE**](detection-type.md)<br/>                                 | The DETECTION\_TYPE enumeration type is used in conjunction with the [**IOCTL\_DISK\_GET\_DRIVE\_GEOMETRY\_EX**](ioctl-disk-get-drive-geometry-ex.md) request and the [**DISK\_GEOMETRY\_EX**](disk-geometry-ex.md) structure to determine the type of formatting used by the BIOS to record the disk geometry. <br/>                                                                                                                                     |
| [**DISK\_CACHE\_INFORMATION**](disk-cache-information.md)<br/>                | The DISK\_CACHE\_INFORMATION structure is used with the [**IOCTL\_DISK\_GET\_CACHE\_INFORMATION**](ioctl-disk-get-cache-information.md) request to retrieve cache information.<br/>                                                                                                                                                                                                                                                                        |
| [**DISK\_CACHE\_RETENTION\_PRIORITY**](disk-cache-retention-priority.md)<br/> | The DISK\_CACHE\_RETENTION\_PRIORITY enumeration is used in conjunction with the [**IOCTL\_DISK\_GET\_CACHE\_INFORMATION**](ioctl-disk-get-cache-information.md) request and the structure [**DISK\_CACHE\_INFORMATION**](disk-cache-information.md) to indicate which kinds data are to be held in the cache on a preferential basis. <br/>                                                                                                              |
| [**DISK\_CONTROLLER\_NUMBER**](disk-controller-number.md)<br/>                | DISK\_CONTROLLER\_NUMBER is used with IOCTL\_DISK\_CONTROLLER\_NUMBER to retrieve the controller number and disk number of an IDE disk. <br/>                                                                                                                                                                                                                                                                                                               |
| [**DISK\_COPY\_DATA\_PARAMETERS**](disk-copy-data-parameters.md)<br/>         | DISK\_COPY\_DATA\_PARAMETERS is used with IOCTL\_DISK\_COPY\_DATA to copy data from one area of the disk to another.<br/>                                                                                                                                                                                                                                                                                                                                   |
| [**DISK\_DETECTION\_INFO**](disk-detection-info.md)<br/>                      | The DISK\_DETECTION\_INFO structure contains the detected drive parameters that are supplied by an x86 PC BIOS on boot.<br/>                                                                                                                                                                                                                                                                                                                                |
| [**DISK\_GEOMETRY**](disk-geometry.md)<br/>                                   | The DISK\_GEOMETRY structure is obsolete and provided only to support existing drivers. New drivers must use [**DISK\_GEOMETRY\_EX**](disk-geometry-ex.md). DISK\_GEOMETRY is used in conjunction with the [**IOCTL\_DISK\_GET\_DRIVE\_GEOMETRY**](ioctl-disk-get-drive-geometry.md) and the [**IOCTL\_DISK\_GET\_MEDIA\_TYPES**](ioctl-disk-get-media-types.md) requests, in order to retrieve information about the geometry of a physical disk. <br/> |
| [**DISK\_GEOMETRY\_EX**](disk-geometry-ex.md)<br/>                            | The [**DISK\_GEOMETRY\_EX**](disk-geometry-ex.md) structure is a variable-length structure composed of a [**DISK\_GEOMETRY**](disk-geometry.md) structure followed by a [**DISK\_PARTITION\_INFO**](disk-partition-info.md) structure followed, in turn, by a [**DISK\_DETECTION\_INFO**](disk-detection-info.md) structure. <br/>                                                                                                                      |
| [**DISK\_GROW\_PARTITION**](disk-grow-partition.md)<br/>                      | The DISK\_GROW\_PARTITION structure is used in conjunction with the [**IOCTL\_DISK\_GROW\_PARTITION**](ioctl-disk-grow-partition.md) request to enlarge a partition. <br/>                                                                                                                                                                                                                                                                                 |
| [**DISK\_INT13\_INFO**](disk-int13-info.md)<br/>                              | The DISK\_INT13\_INFO structure is used by the BIOS to report disk detection data for a partition with an INT13 format.<br/>                                                                                                                                                                                                                                                                                                                                |
| [**DISK\_EX\_INT13\_INFO**](disk-ex-int13-info.md)<br/>                       | The DISK\_EX\_INT13\_INFO structure is used by the BIOS to report disk detection data for a partition with an extended INT13 format. <br/>                                                                                                                                                                                                                                                                                                                  |
| [**DISK\_PARTITION\_INFO**](disk-partition-info.md)<br/>                      | The [**DISK\_PARTITION\_INFO**](disk-partition-info.md) structure is used to report information about the disk's partition table. <br/>                                                                                                                                                                                                                                                                                                                    |
| [**DISK\_PERFORMANCE**](disk-performance.md)<br/>                             | The DISK\_PERFORMANCE structure is used in conjunction with the [**IOCTL\_DISK\_PERFORMANCE**](ioctl-disk-performance.md) request to collect summary disk statistics for purposes of measuring disk performance. <br/>                                                                                                                                                                                                                                     |
| [**DRIVE\_LAYOUT\_INFORMATION**](drive-layout-information.md)<br/>            | The DRIVE\_LAYOUT\_INFORMATION structure is obsolete and is provided only to support existing drivers. New drivers must use [**DRIVE\_LAYOUT\_INFORMATION\_EX**](drive-layout-information-ex.md). <br/> The DRIVE\_LAYOUT\_INFORMATION structure is used to report information about a disk drive and its partitions. It is also used to write new drive layout information to the disk. <br/>                                                       |
| [**DRIVE\_LAYOUT\_INFORMATION\_EX**](drive-layout-information-ex.md)<br/>     | The DRIVE\_LAYOUT\_INFORMATION\_EX structure is used to report information about the driver layout. <br/>                                                                                                                                                                                                                                                                                                                                                   |
| [**DRIVE\_LAYOUT\_INFORMATION\_GPT**](drive-layout-information-gpt.md)<br/>   | The DRIVE\_LAYOUT\_INFORMATION\_GPT structure reports the drive signature for a GUID Partition Table partition. <br/>                                                                                                                                                                                                                                                                                                                                       |
| [**DRIVE\_LAYOUT\_INFORMATION\_MBR**](drive-layout-information-mbr.md)<br/>   | The DRIVE\_LAYOUT\_INFORMATION\_MBR structure reports the drive signature for a Master Boot Record partition. <br/>                                                                                                                                                                                                                                                                                                                                         |
| [**DRIVERSTATUS**](driverstatus.md)<br/>                                      | The DRIVERSTATUS structure is used in conjunction with the [**SENDCMDOUTPARAMS**](sendcmdoutparams.md) structure and the [**SMART\_SEND\_DRIVE\_COMMAND**](smart-send-drive-command.md) request to retrieve data returned by a Self-Monitoring Analysis and Reporting Technology (SMART) command.<br/>                                                                                                                                                    |
| [**FORMAT\_EX\_PARAMETERS**](format-ex-parameters.md)<br/>                    | The FORMAT\_EX\_PARAMETERS structure is used in conjunction with the [**IOCTL\_DISK\_FORMAT\_TRACKS\_EX**](ioctl-disk-format-tracks-ex.md) request to format the specified set of contiguous tracks on the disk. <br/>                                                                                                                                                                                                                                     |
| [**FORMAT\_PARAMETERS**](format-parameters.md)<br/>                           | The FORMAT\_PARAMETERS structure is used in conjunction with the [**IOCTL\_DISK\_FORMAT\_TRACKS**](ioctl-disk-format-tracks.md) request to format the specified set of contiguous tracks on the disk. <br/>                                                                                                                                                                                                                                                |
| [**GET\_LENGTH\_INFORMATION**](get-length-information.md)<br/>                | The GET\_LENGTH\_INFORMATION structure is used with the [**IOCTL\_DISK\_GET\_LENGTH\_INFO**](ioctl-disk-get-length-info.md) to obtain the length, in bytes, of a disk, partition, or volume. <br/>                                                                                                                                                                                                                                                         |
| [**GETVERSIONINPARAMS**](getversioninparams.md)<br/>                          | The GETVERSIONINPARAMS structure is used in conjunction with the [**SMART\_GET\_VERSION**](smart-get-version.md) request to retrieve version information, a capabilities mask, and a bitmask for the indicated device. <br/>                                                                                                                                                                                                                               |
| [**IDEREGS**](ideregs.md)<br/>                                                | The IDEREGS structure is used to report the contents of the IDE controller registers. <br/>                                                                                                                                                                                                                                                                                                                                                                 |
| [**MEDIA\_TYPE**](media-type.md)<br/>                                         | The MEDIA\_TYPE enumerators are used in conjunction with the [**IOCTL\_DISK\_FORMAT\_TRACKS**](ioctl-disk-format-tracks.md) request to format the specified set of contiguous tracks on the disk. <br/>                                                                                                                                                                                                                                                    |
| [**PARTITION\_INFORMATION**](partition-information.md)<br/>                   | The PARTITION\_INFORMATION structure contains partition information for a partition with a traditional AT-style Master Boot Record (MBR). <br/>                                                                                                                                                                                                                                                                                                             |
| [**PARTITION\_INFORMATION\_EX**](partition-information-ex.md)<br/>            | PARTITION\_INFORMATION\_EX is the extended version of the [**PARTITION\_INFORMATION**](partition-information.md) structure. It holds information both for partitions with a Master Boot Record and for partitions with a GUID Partition Table.<br/>                                                                                                                                                                                                        |
| [**PARTITION\_INFORMATION\_GPT**](partition-information-gpt.md)<br/>          | PARTITION\_INFORMATION\_GPT contains information for a GUID Partition Table partition that is not held in common with a Master Boot Record partition.<br/>                                                                                                                                                                                                                                                                                                  |
| [**PARTITION\_INFORMATION\_MBR**](partition-information-mbr.md)<br/>          | PARTITION\_INFORMATION\_MBR contains information for a Master Boot Record partition that is not held in common with a GUID Partition Table partition.<br/>                                                                                                                                                                                                                                                                                                  |
| [**PARTITION\_STYLE**](partition-style.md)<br/>                               | The PARTITION\_STYLE enumeration type indicates the type of partition table for a disk.<br/>                                                                                                                                                                                                                                                                                                                                                                |
| [**REASSIGN\_BLOCKS**](reassign-blocks.md)<br/>                               | The **REASSIGN\_BLOCKS** structure is used in conjunction with the [**IOCTL\_DISK\_REASSIGN\_BLOCKS**](ioctl-disk-reassign-blocks.md) request to instruct a disk device to reassign the block numbers of the indicated bad blocks to good blocks.<br/>                                                                                                                                                                                                     |
| [**REASSIGN\_BLOCKS\_EX**](reassign-blocks-ex.md)<br/>                        | The **REASSIGN\_BLOCKS\_EX** structure is used in conjunction with the [**IOCTL\_DISK\_REASSIGN\_BLOCKS\_EX**](ioctl-disk-reassign-blocks-ex.md) request to instruct a disk device to reassign the block numbers of the indicated bad blocks to good blocks.<br/>                                                                                                                                                                                          |
| [**SENDCMDINPARAMS**](sendcmdinparams.md)<br/>                                | The SENDCMDINPARAMS structure contains the input parameters for the [**SMART\_SEND\_DRIVE\_COMMAND**](smart-send-drive-command.md) request.<br/>                                                                                                                                                                                                                                                                                                           |
| [**SENDCMDOUTPARAMS**](sendcmdoutparams.md)<br/>                              | The SENDCMDOUTPARAMS structure is used in conjunction with the [**SMART\_SEND\_DRIVE\_COMMAND**](smart-send-drive-command.md) request to retrieve data returned by a Self-Monitoring Analysis and Reporting Technology (SMART) command. <br/>                                                                                                                                                                                                              |
| [**SET\_PARTITION\_INFORMATION**](set-partition-information.md)<br/>          | SET\_PARTITION\_INFORMATION is used with IOCTL\_DISK\_SET\_PARTITION\_INFO to change the partition type of a specified Master Boot Record (MBR) disk partition.<br/>                                                                                                                                                                                                                                                                                        |
| [**SET\_PARTITION\_INFORMATION\_EX**](set-partition-information-ex.md)<br/>   | **SET\_PARTITION\_INFORMATION\_EX** is used with the IOCTL [**IOCTL\_DISK\_SET\_PARTITION\_INFO\_EX**](ioctl-disk-set-partition-info-ex.md) to set information for a specific partition.<br/>                                                                                                                                                                                                                                                              |
| [**VERIFY\_INFORMATION**](verify-information.md)<br/>                         | The VERIFY\_INFORMATION structure provides information used to verify the existence of a disk extent. <br/>                                                                                                                                                                                                                                                                                                                                                 |
