<dl> <dt>

<span id="mmc_data_object_gly"></span><span id="MMC_DATA_OBJECT_GLY"></span>**data object**
</dt> <dd>

For COM. Used for data transfer between MMC and snap-ins and between multiple snap-ins. A data object is an instance of a class that implements the **IDataObject** interface. MMC requests data objects from snap-ins that insert scope or result items by calling the **IComponentData::QueryDataObject** or **IComponent::QueryDataObject** method.

</dd> <dt>

<span id="mmc_default_list_view_taskpad_gly"></span><span id="MMC_DEFAULT_LIST_VIEW_TASKPAD_GLY"></span>**default list view taskpad**
</dt> <dd>

See [*standard list view taskpad*](https://www.bing.com/search?q=*standard list view taskpad*).

</dd> <dt>

<span id="mmc_default_taskpad_gly"></span><span id="MMC_DEFAULT_TASKPAD_GLY"></span>**default taskpad**
</dt> <dd>

See [*standard taskpad*](https://www.bing.com/search?q=*standard taskpad*).

</dd> <dt>

<span id="mmc_default_verb_gly"></span><span id="MMC_DEFAULT_VERB_GLY"></span>**default verb**
</dt> <dd>

[*Verb*](https://www.bing.com/search?q=*Verb*) that specifies the default action for a scope item or result item when the user double-clicks or presses ENTER on that item. Default verbs appear in bold type.

</dd> <dt>

<span id="mmc_designer_window_gly"></span><span id="MMC_DESIGNER_WINDOW_GLY"></span>**designer window**
</dt> <dd>

Window in the Snap-in Designer for Visual Basic in which snap-in developers can define most of the snap-in's user interface and runtime behavior.

</dd> <dt>

<span id="mmc_details_pane_gly"></span><span id="MMC_DETAILS_PANE_GLY"></span>**details pane**
</dt> <dd>

Used in the MMC end-user documentation to refer to the [*result pane*](https://www.bing.com/search?q=*result pane*).

</dd> <dt>

<span id="mmc_details_view_gly"></span><span id="MMC_DETAILS_VIEW_GLY"></span>**details view**
</dt> <dd>

Used in the MMC end-user documentation to refer to the [*result pane*](https://www.bing.com/search?q=*result pane*).

</dd> <dt>

<span id="mmc_dialog_unit_gly"></span><span id="MMC_DIALOG_UNIT_GLY"></span>**dialog unit**
</dt> <dd>

Unit of measurement in Visual C++ dialog box templates to determine the size of dialog boxes. Also referred to as "dialog box units."

</dd> <dt>

<span id="mmc_dual_mode_snap_in_gly"></span><span id="MMC_DUAL_MODE_SNAP_IN_GLY"></span>**dual-mode snap-in**
</dt> <dd>

Snap-in that operates both as a [*stand-alone snap-in*](https://www.bing.com/search?q=*stand-alone snap-in*) and as an [*extension snap-in*](https://www.bing.com/search?q=*extension snap-in*). An example of a dual-mode snap-in is the Event Viewer snap-in.

</dd> <dt>

<span id="mmc_dynamic_extension_gly"></span><span id="MMC_DYNAMIC_EXTENSION_GLY"></span>**dynamic extension**
</dt> <dd>

[*Extension snap-in*](https://www.bing.com/search?q=*Extension snap-in*) that is added to a scope or result item programmatically at run time. A [*primary snap-in*](https://www.bing.com/search?q=*primary snap-in*) can dynamically add any type of extension to any of its own items.

</dd> <dt>

<span id="mmc_dynamic_link_library_dll__gly"></span><span id="MMC_DYNAMIC_LINK_LIBRARY_DLL__GLY"></span>**dynamic-link library (DLL)**
</dt> <dd>

Modules that contains functions and data that can be loaded at runtime by a calling module (an executable file or another DLL). When a DLL is loaded, it is mapped into the address space of the calling process (module). The calling module can call the exported functions of the DLL to make use of the functionality provided by the DLL.

[*Snap-ins*](https://www.bing.com/search?q=*Snap-ins*) are implemented as in-process server DLLs that run in the process space of an instance of MMC.

</dd> </dl>
