# M

[A](a-gly.md) [B](b-gly.md) [C](c-gly.md) [D](d-gly.md) [E](e-gly.md) [F](f-gly.md) [G](g-gly.md) [H](h-gly.md) [I](i-gly.md) J K [L](l-gly.md) M [N](n-gly.md) [O](o-gly.md) [P](p-gly.md) Q [R](r-gly.md) [S](s-gly.md) [T](t-gly.md) [U](u-gly.md) [V](v-gly.md) [W](w-gly.md) X Y Z

<dl> <dt>

<span id="mmc_main_mmc_window_gly"></span><span id="MMC_MAIN_MMC_WINDOW_GLY"></span>**main MMC window**
</dt> <dd>

A [*Multiple-document interface (MDI)*](#mmc-multiple-document-interface-mdi--gly) parent window that appears when MMC is started.

</dd> <dt>

<span id="mmc_manually_created_node_gly"></span><span id="MMC_MANUALLY_CREATED_NODE_GLY"></span>**manually created node**
</dt> <dd>

A node in a Visual Basic snap-in that is defined at design time and is explicitly created in the console by Visual Basic code in the snap-in.

</dd> <dt>

<span id="mmc_menu_bar_gly"></span><span id="MMC_MENU_BAR_GLY"></span>**menu bar**
</dt> <dd>

The area of the main MMC window below the title bar that contains pop-up menus such as Console, Window, and Help.

</dd> <dt>

<span id="mmc_menu_button_gly"></span><span id="MMC_MENU_BUTTON_GLY"></span>**menu button**
</dt> <dd>

A pop-up menu on the main MMC window that is added either by MMC, a [*stand-alone*](https://www.bing.com/search?q=*stand-alone*), or an [*extension snap-in*](https://www.bing.com/search?q=*extension snap-in*).

</dd> <dt>

<span id="mmc_menu_button_control_gly"></span><span id="MMC_MENU_BUTTON_CONTROL_GLY"></span>**menu button control**
</dt> <dd>

An MMC control for managing pop-up menus in the main MMC window.

</dd> <dt>

<span id="mmc_message_view_gly"></span><span id="MMC_MESSAGE_VIEW_GLY"></span>**Message View**
</dt> <dd>

An [*ActiveX control*](https://www.bing.com/search?q=*ActiveX control*) supplied by MMC that displays an icon, a title, and message text.

</dd> <dt>

<span id="mmc_modal_gly"></span><span id="MMC_MODAL_GLY"></span>**modal**
</dt> <dd>

A dialog box to which the user must respond before continuing work. A modal dialog box can be either [*application modal*](https://www.bing.com/search?q=*application modal*) or [*system modal*](https://www.bing.com/search?q=*system modal*).

</dd> <dt>

<span id="mmc_modeless_gly"></span><span id="MMC_MODELESS_GLY"></span>**modeless**
</dt> <dd>

A dialog box to which the user is not required to respond before continuing work. In MMC, property sheets displayed by snap-ins are modeless.

</dd> <dt>

<span id="mmc_msi_gly"></span><span id="MMC_MSI_GLY"></span>**MSI**
</dt> <dd>

Also see [Microsoft Installer](microsoft-installer-integration.md).

</dd> <dt>

<span id="mmc_msi_package_gly"></span><span id="MMC_MSI_PACKAGE_GLY"></span>**MSI package**
</dt> <dd>

An installation package contains all the information that MSI requires to install or uninstall an application or product and to run the setup user interface. Each installation package includes an .msi file, which contains an installation database, a summary information stream, and data streams for various parts of the installation. The .msi file can also contain one or more transforms, internal source files, and external source files or cabinet files required by the installation.

</dd> <dt>

<span id="mmc_multiple_document_interface_mdi__gly"></span><span id="MMC_MULTIPLE_DOCUMENT_INTERFACE_MDI__GLY"></span>**multiple-document interface (MDI)**
</dt> <dd>

An interface that defines a user interface for applications that enable the user to work with more than one document simultaneously. Each document in an MDI application is displayed in a separate child window within the client area of the application's main window.

</dd> <dt>

<span id="mmc_multiselection_data_object_gly"></span><span id="MMC_MULTISELECTION_DATA_OBJECT_GLY"></span>**multiselection data object**
</dt> <dd>

A data object that a snap-in creates when it receives the MMC\_MULTI\_SELECT\_COOKIE special cookie value in a call to its [**IComponent::QueryDataObject**](/windows/desktop/api/Mmc/nf-mmc-icomponent-querydataobject) method. MMC requests a multiselection data object during a multiple selection. This data object should supply the list of node types for all of the currently selected result items owned by the snap-in in the multiple selection.

</dd> </dl>
