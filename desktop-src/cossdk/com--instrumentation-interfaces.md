# COM+ Instrumentation Interfaces

The COM+ Instrumentation metrics are published to a client by using [COM+ Events](com--events.md), a loosely coupled events (LCE) system that stores event information from different publishers in an event store in the COM+ catalog. The metrics are grouped in categories based on the type of object for which they provide information. For each category, there is a correspondent interface to which the client can subscribe, and the subscribing client receives all the metrics in that category. The client can also subscribe to specific methods on the interface to receive specific metrics through filtering metrics.

The COM+ instrumentation interfaces described in the following table allow clients to receive notifications for a variety of COM+ instrumentation metrics.



| Interfaces                                                                                                                        | Metrics provided                                                                                                                                                                                                                                                                                                          |
|-----------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**IComActivityEvents**](icomactivityevents.md)<br/>                                                                       | The user implements [**IComActivityEvents**](icomactivityevents.md) to be notified when an activity is created, destroyed, or timed out.<br/>                                                                                                                                                                      |
| [**IComAppEvents**](icomappevents.md)<br/>                                                                                 | The user implements [**IComAppEvents**](icomappevents.md) to be notified when a server application starts, shuts down, or is forced to shut down. <br/>                                                                                                                                                            |
| [**IComApp2Events**](icomapp2events.md)<br/>                                                                               | The user implements [**IComApp2Events**](icomapp2events.md) to be notified if a COM+ server application is loaded, shut down, or paused. The subscriber is also notified if the application is marked for recycling.<br/>                                                                                          |
| [**IComCRMEvents**](icomcrmevents.md)<br/>                                                                                 | The user implements [**IComCRMEvents**](icomcrmevents.md) to be notified when the Compensating Resource Manager (CRM) recovery starts or completes. <br/>                                                                                                                                                          |
| [**IComExceptionEvents**](icomexceptionevents.md)<br/>                                                                     | The user implements [**IComExceptionEvents**](icomexceptionevents.md) to be notified when a user exception has occurred in a transactional component. <br/>                                                                                                                                                        |
| [**IComIdentityEvents**](icomidentityevents.md)<br/>                                                                       | The [**IComIdentityEvents**](icomidentityevents.md) interface notifies the subscriber of an activity that is part of an Internet Information Services (IIS) Active Server Pages (ASP) page.<br/>                                                                                                                   |
| [**IComInstanceEvents**](icominstanceevents.md) <br/> [**IComInstance2Events**](icominstance2events.md)<br/>        | The [**IComInstanceEvents**](icominstanceevents.md) or [**IComInstance2Events**](icominstance2events.md) interface notifies the subscriber of an object's creation or release.<br/>                                                                                                                               |
| [**IComMethodEvents**](icommethodevents.md) <br/> [**IComMethod2Events**](icommethod2events.md)<br/>                | The user implements [**IComMethodEvents**](icommethodevents.md) or [**IComMethod2Events**](icommethod2events.md) to be notified when an object's method is called, returned, or generates an exception.<br/>                                                                                                      |
| [**IComObjectConstructionEvents**](icomobjectconstructionevents.md)<br/>                                                   | The [**IComObjectConstructionEvents**](icomobjectconstructionevents.md) interface notifies the subscriber if a constructed object is created in an object pool.<br/>                                                                                                                                               |
| [**IComObjectConstruction2Events**](icomobjectconstruction2events.md)<br/>                                                 | The [**IComObjectConstruction2Events**](icomobjectconstruction2events.md) interface notifies the subscriber if a constructed object is created.<br/>                                                                                                                                                               |
| [**IComObjectEvents**](icomobjectevents.md)<br/>                                                                           | The user implements [**IComObjectEvents**](icomobjectevents.md) to be notified when a JIT-activated object is created or released.<br/>                                                                                                                                                                            |
| [**IComObjectPoolEvents**](icomobjectpoolevents.md)<br/> [**IComObjectPool2Events**](icomobjectpool2events.md)<br/> | The user implements [**IComObjectPoolEvents**](icomobjectpoolevents.md) or [**IComObjectPool2Events**](icomobjectpool2events.md) to be notified if a transactional or non-transactional object is added to or obtained from the object pool. <br/>                                                                |
| [**IComObjectPoolEvents2**](icomobjectpoolevents2.md)<br/>                                                                 | The user implements [**IComObjectPoolEvents2**](icomobjectpoolevents2.md) to be notified when a new pool is created or an object is returned or obtained from the pool.<br/>                                                                                                                                       |
| [**IComQCEvents**](icomqcevents.md)<br/>                                                                                   | The user implements [**IComQCEvents**](icomqcevents.md) to be notified when a queued message is created or de-queued.<br/>                                                                                                                                                                                         |
| [**IComResourceEvents**](icomresourceevents.md)<br/>                                                                       | The user implements [**IComResourceEvents**](icomresourceevents.md) to be notified when a new resource is created, allocated, destroyed, or tracked.<br/>                                                                                                                                                          |
| [**IComSecurityEvents**](icomsecurityevents.md)<br/>                                                                       | The [**IComSecurityEvents**](icomsecurityevents.md) interface notifies the subscriber if the authentication of a method call succeeded or failed.<br/>                                                                                                                                                             |
| [**IComThreadEvents**](icomthreadevents.md)<br/>                                                                           | The [**IComThreadEvents**](icomthreadevents.md) interface notifies the subscriber if a single-threaded apartment (STA) is created or terminated, and when an apartment thread is allocated. The subscriber is also notified if an activity is assigned or unassigned to an apartment thread.<br/>                  |
| [**IComTrackingInfoCollection**](icomtrackinginfocollection.md)<br/>                                                       | The [**IComTrackingInfoCollection**](icomtrackinginfocollection.md) interface notifies the subscriber of the type and count of tracked collections.<br/>                                                                                                                                                           |
| [**IComTrackingInfoEvents**](icomtrackinginfoevents.md)<br/>                                                               | The [**IComTrackingInfoEvents**](icomtrackinginfoevents.md) interface notifies the subscriber of tracked events.<br/>                                                                                                                                                                                              |
| [**IComTrackingInfoObject**](icomtrackinginfoobject.md)<br/>                                                               | The [**IComTrackingInfoObject**](icomtrackinginfoobject.md) interface notifies the subscriber of changes to the properties of tracked objects.<br/>                                                                                                                                                                |
| [**IComTrackingInfoProperties**](icomtrackinginfoproperties.md)<br/>                                                       | The [**IComTrackingInfoProperties**](icomtrackinginfoproperties.md) interface notifies the subscriber of changes to the count and names of tracked properties.<br/>                                                                                                                                                |
| [**IComTransactionEvents**](icomtransactionevents.md)<br/>                                                                 | The user implements [**IComTransactionEvents**](icomtransactionevents.md) to be notified when a distributed transaction of a component is committed or aborted.<br/>                                                                                                                                               |
| [**IComTransaction2Events**](icomtransaction2events.md)<br/>                                                               | The [**IComTransaction2Events**](icomtransaction2events.md) interface notifies the subscriber if a Microsoft Distributed Transaction Coordinator (DTC) transaction starts, commits, or aborts. The subscriber is also notified when the transaction is in the prepare phase of the two-phase commit protocol.<br/> |
| [**IComUserEvent**](icomuserevent.md)<br/>                                                                                 | The user implements [**IComUserEvent**](icomuserevent.md) to be notified when a user-defined event has occurred.<br/>                                                                                                                                                                                              |
| [**ISystemAppEventData**](isystemappeventdata.md)<br/>                                                                     | The [**ISystemAppEventData**](isystemappeventdata.md) interface notifies the subscriber of the creation and modification of COM+ application instances.<br/>                                                                                                                                                       |
