# Characteristics of Attributes

Each attribute in an Active Directory Domain Services object is defined by an [**attributeSchema**](https://msdn.microsoft.com/library/ms680969) object in the schema container. The properties of an **attributeSchema** object specify the characteristics of the attribute, such as:

-   Attribute identifiers. Attributes have several identifiers, of which the most interesting from a programming perspective are the [**lDAPDisplayName**](https://msdn.microsoft.com/library/ms676828), which is used by LDAP clients to read and write the attribute, and the [**schemaIDGUID**](https://msdn.microsoft.com/library/ms679645), which is used in security descriptors to control access to the attribute.
-   The type of data contained by instances of the attribute. An attribute's syntax properties determine the type of data, such as integer, string, or binary. Additional properties can specify the range of values that are allowed for the attribute and whether an instance of the attribute can have multiple values.
-   Including the attribute in groups. Other properties tag an attribute to be included in a property set, which is a set of related properties, or to be included in the set of attributes that are replicated in the global catalog or indexed to optimize search performance.

The following table lists the [**lDAPDisplayName**](https://msdn.microsoft.com/library/ms676828) and description of the key properties of an [**attributeSchema**](https://msdn.microsoft.com/library/ms680969) object. For more information and a complete list of the mandatory and optional properties of an **attributeSchema** object, see [**Attribute-Schema Class**](https://msdn.microsoft.com/library/ms680969).

| [**cn**](https://msdn.microsoft.com/library/ms675449)<br/>                                                       | Every object in Active Directory Domain Services has a naming attribute from which its RDN is formed. | [**rangeLower**](https://msdn.microsoft.com/library/ms679437), [**rangeUpper**](https://msdn.microsoft.com/library/ms679438)         | A pair of integers that specify the lower and upper bounds of the range of values for this attribute. For attributes with binary syntax, the range specifies the number of bytes.<br/> If both [**rangeLower**](https://msdn.microsoft.com/library/ms679437) and [**rangeUpper**](https://msdn.microsoft.com/library/ms679438) are set, **rangeLower** must be less than **rangeUpper**. For example, if the **rangeLower** for an integer is 3, and **rangeUpper** is absent, it means there is no upper constraint on the attribute. Likewise, if rangeUpper for a string is 2000, and rangeLower is absent, this indicates that there is no lower constraint on the length.<br/> |
