# Appendix D: Notes for Visual Basic Developers

This section provides information about Microsoft Active Accessibility for Microsoft Visual Basic developers.

Applications written in Visual Basic are Microsoft Active Accessibility clients. They do not provide information about their custom user interface elements because they do not implement [**IAccessible**](iaccessible.md) or any other Component Object Model (COM) interface.

This documentation uses the C/C++ names for the [**IAccessible**](iaccessible.md) properties; however, the Visual Basic names are similar. For example, the [**IAccessible::get\_accName**](iaccessible-iaccessible--get-accname.md) property would be called **accName** in a Visual Basic application.
