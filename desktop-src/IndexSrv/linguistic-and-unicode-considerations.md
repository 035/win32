# Linguistic and Unicode Considerations

> [!Note]  
> Indexing Service is no longer supported as of Windows XP and is unavailable for use as of Windows 8. Instead, use [Windows Search](https://msdn.microsoft.com/library/windows/desktop/aa965362) for client side search and [Microsoft Search Server Express]( http://go.microsoft.com/fwlink/p/?linkid=258445) for server side search.

 

This section contains a list of linguistic and Unicode considerations that might affect word breaker and stemmer implementation. The list is not an exhaustive one.

This section includes the following topics:

-   [Surface Form Normalization](surface-form-normalization.md) describes surface forms that word breakers and stemmers may normalize to.
-   [Phrase Identification](phrase-identification.md) describes how word breakers identify phrases in text.
-   [Agglutinative Languages](agglutinative-languages.md) describes stemming considerations for agglutinative languages.
-   [Numbers and Dates](numbers-and-dates.md) describes how word breakers and stemmers handle numbers and dates.
-   [Compound Words](compound-words.md) describes how word breakers and stemmers handle compound words.
-   [Compound Phrases](compound-phrases.md) describes how word breakers and stemmers handle compound phrases.
-   [Special Characters and Words](special-characters-and-words.md) describes how word breakers and stemmers handle special words and characters.
-   [Acronyms and Abbreviations](acronyms-and-abbreviations.md) describes how word breakers and stemmers handle acronyms and abbreviations.
-   [Capitalization](capitalization.md) describes how word breakers and stemmers handle capitalization.
-   [Nonbreaking Spaces](nonbreaking-spaces.md) describes how word breakers handle nonbreaking spaces.
-   [Surrogate Pairs](surrogate-pairs.md) describes Unicode surrogate pairs and using surrogate pairs to extend the Unicode character set to accommodate different character sets.
