# Task Scheduler 2.0 Enumerated Types

Describes the enumerated types that define the constants that are used by the Task Scheduler 2.0 APIs.

## 

The following enumerated types are introduced in Task Scheduler 2.0.



| Enumeration                                                                  | Description                                                                                                      |
|------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------|
| [**TASK\_ACTION\_TYPE**](task-action-type.md)                               | Defines the type of actions that a task can perform.                                                             |
| [**TASK\_COMPATIBILITY**](task-compatibility.md)                            | Defines what versions of Task Scheduler or the AT command that the task is compatible with.                      |
| [**TASK\_CREATION**](taskcreation.md)                                       | Defines how the Task Scheduler service creates, updates, or disables the task.                                   |
| [**TASK\_ENUM\_FLAGS**](task-enum-flags.md)                                 | Defines how the Task Scheduler enumerates through registered tasks.                                              |
| [**TASK\_INSTANCES\_POLICY**](task-instances-policy.md)                     | Defines how the Task Scheduler handles existing instances of the task when it starts a new instance of the task. |
| [**TASK\_LOGON TYPE**](task-logon-type.md)                                  | Defines what logon technique is required to run a task.                                                          |
| [**TASK\_PROCESSTOKENSID\_TYPE**](task-processtokensid-type.md)             | Defines the types of process SID that can be used by tasks.                                                      |
| [**TASK\_RUN\_FLAGS**](task-run-flags.md)                                   | Defines how a task is run.                                                                                       |
| [**TASK\_RUNLEVEL\_TYPE**](task-runlevel-type.md)                           | Defines LUA elevation flags that specify with what privilege level the task will be run.                         |
| [**TASK\_SESSION\_STATE\_CHANGE\_TYPE**](task-session-state-change-type.md) | Defines what kinds of Terminal Server session state change you can use to trigger a task to start.               |
| [**TASK\_STATE**](task-state.md)                                            | Defines the different states that a registered task can be in.                                                   |
| [**TASK\_TRIGGER\_TYPE2**](triggertype.md)                                  | Defines the type of triggers that can be used by tasks.                                                          |
