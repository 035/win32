# Usbuser.h

This section contains reference topics for the Usbuser.h header.

## In this section



| Topic                                                                                       | Description                                                                                                                                                                                                                                                                                                                                                          |
|---------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**IOCTL\_USB\_USER\_REQUEST**](/windows/desktop/api/Usbuser/ni-usbuser-ioctl_usb_user_request)<br/>                      | The [**IOCTL\_USB\_USER\_REQUEST**](/windows/desktop/api/Usbuser/ni-usbuser-ioctl_usb_user_request) I/O control request is available to both user-mode applications and kernel-mode drivers. <br/>                                                                                                                                                                                                 |
| [**USB\_BANDWIDTH\_INFO**](/windows/desktop/api/usbuser/ns-usbuser-_usb_bandwidth_info)<br/>                               | The [**USB\_BANDWIDTH\_INFO**](/windows/desktop/api/usbuser/ns-usbuser-_usb_bandwidth_info) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](/windows/desktop/api/Usbuser/ni-usbuser-ioctl_usb_user_request) I/O control request to retrieve information about the allocated bandwidth.<br/>                                                                                                                                  |
| [**USB\_BUS\_STATISTICS\_0**](/windows/desktop/api/usbuser/ns-usbuser-_usb_bus_statistics_0)<br/>                          | The [**USB\_BUS\_STATISTICS\_0**](/windows/desktop/api/usbuser/ns-usbuser-_usb_bus_statistics_0) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](/windows/desktop/api/Usbuser/ni-usbuser-ioctl_usb_user_request) I/O control request to retrieve bus statistics.<br/>                                                                                                                                                        |
| [**USB\_CLOSE\_RAW\_DEVICE\_PARAMETERS**](/windows/desktop/api/usbuser/ns-usbuser-_usb_close_raw_device_parameters)<br/>   | This structure is not supported. <br/> The [**USB\_CLOSE\_RAW\_DEVICE\_PARAMETERS**](/windows/desktop/api/usbuser/ns-usbuser-_usb_close_raw_device_parameters) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](/windows/desktop/api/Usbuser/ni-usbuser-ioctl_usb_user_request) I/O control request to close raw access to devices on the bus.<br/>                                                                     |
| [**USB\_CONTROLLER\_INFO\_0**](/windows/desktop/api/usbuser/ns-usbuser-_usb_controller_info_0)<br/>                        | The [**USB\_CONTROLLER\_INFO\_0**](/windows/desktop/api/usbuser/ns-usbuser-_usb_controller_info_0) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](/windows/desktop/api/Usbuser/ni-usbuser-ioctl_usb_user_request) I/O control request to retrieve information about the USB host controller.<br/>                                                                                                                           |
| [**USB\_DRIVER\_VERSION\_PARAMETERS**](/windows/desktop/api/usbuser/ns-usbuser-_usb_driver_version_parameters)<br/>        | The [**USB\_DRIVER\_VERSION\_PARAMETERS**](/windows/desktop/api/usbuser/ns-usbuser-_usb_driver_version_parameters) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](/windows/desktop/api/Usbuser/ni-usbuser-ioctl_usb_user_request) I/O control request to retrieve version information.<br/>                                                                                                                                 |
| [**USB\_PASS\_THRU\_PARAMETERS**](/windows/desktop/api/usbuser/ns-usbuser-_usb_pass_thru_parameters)<br/>                  | The [**USB\_PASS\_THRU\_PARAMETERS**](/windows/desktop/api/usbuser/ns-usbuser-_usb_pass_thru_parameters) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](/windows/desktop/api/Usbuser/ni-usbuser-ioctl_usb_user_request) I/O control request to pass a vendor-specific command to the host controller miniport driver.<br/>                                                                                                  |
| [**USB\_POWER\_INFO**](/windows/desktop/api/usbuser/ns-usbuser-_usb_power_info)<br/>                                       | The [**USB\_POWER\_INFO**](/windows/desktop/api/usbuser/ns-usbuser-_usb_power_info) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](/windows/desktop/api/Usbuser/ni-usbuser-ioctl_usb_user_request) I/O control request to retrieve device power state that the host controller power policy specifies for the indicated system power state.<br/>                                                                            |
| [**USB\_UNICODE\_NAME**](/windows/desktop/api/usbuser/ns-usbuser-_usb_unicode_name)<br/>                                   | The [**USB\_UNICODE\_NAME**](/windows/desktop/api/usbuser/ns-usbuser-_usb_unicode_name) structure contains a Unicode string that specifies a symbolic link name.<br/>                                                                                                                                                                                                                               |
| [**USB\_USER\_ERROR\_CODE**](/windows/desktop/api/usbuser/ne-usbuser-_usb_user_error_code)<br/>                            | The [**USB\_USER\_ERROR\_CODE**](/windows/desktop/api/usbuser/ne-usbuser-_usb_user_error_code) enumeration lists the error codes that a USB user-mode request reports when it fails.<br/>                                                                                                                                                                                                           |
| [**USBUSER\_BANDWIDTH\_INFO\_REQUEST**](/windows/desktop/api/usbuser/ns-usbuser-_usbuser_bandwidth_info_request)<br/>      | The [**USBUSER\_BANDWIDTH\_INFO\_REQUEST**](/windows/desktop/api/usbuser/ns-usbuser-_usbuser_bandwidth_info_request) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](/windows/desktop/api/Usbuser/ni-usbuser-ioctl_usb_user_request) I/O control request to retrieve information about the allocated bandwidth.<br/>                                                                                                         |
| [**USBUSER\_BUS\_STATISTICS\_0\_REQUEST**](/windows/desktop/api/usbuser/ns-usbuser-_usbuser_bus_statistics_0_request)<br/> | The [**USBUSER\_BUS\_STATISTICS\_0\_REQUEST**](/windows/desktop/api/usbuser/ns-usbuser-_usbuser_bus_statistics_0_request) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](/windows/desktop/api/Usbuser/ni-usbuser-ioctl_usb_user_request) I/O control request to retrieve bus statistics.<br/>                                                                                                                               |
| [**USBUSER\_CONTROLLER\_UNICODE\_NAME**](/windows/desktop/api/usbuser/ns-usbuser-_usbuser_controller_unicode_name)<br/>    | The [**USBUSER\_CONTROLLER\_UNICODE\_NAME**](/windows/desktop/api/usbuser/ns-usbuser-_usbuser_controller_unicode_name) structure is used in conjunction with the [**IOCTL\_USB\_USER\_REQUEST**](/windows/desktop/api/Usbuser/ni-usbuser-ioctl_usb_user_request) I/O control request to retrieve the USB host controller driverkey name.<br/>                                                                                           |
| [**USBUSER\_GET\_DRIVER\_VERSION**](/windows/desktop/api/usbuser/ns-usbuser-_usbuser_get_driver_version)<br/>              | The [**USBUSER\_GET\_DRIVER\_VERSION**](/windows/desktop/api/usbuser/ns-usbuser-_usbuser_get_driver_version) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](/windows/desktop/api/Usbuser/ni-usbuser-ioctl_usb_user_request) I/O control request to read driver and interface version information.<br/>                                                                                                                      |
| [**USBUSER\_PASS\_THRU\_REQUEST**](/windows/desktop/api/usbuser/ns-usbuser-_usbuser_pass_thru_request)<br/>                | The [**USBUSER\_PASS\_THRU\_REQUEST**](/windows/desktop/api/usbuser/ns-usbuser-_usbuser_pass_thru_request) structure is used in conjunction with the [**IOCTL\_USB\_USER\_REQUEST**](/windows/desktop/api/Usbuser/ni-usbuser-ioctl_usb_user_request) I/O control request to send a vendor-specific command to the host controller miniport driver.<br/>                                                                                 |
| [**USBUSER\_POWER\_INFO\_REQUEST**](/windows/desktop/api/usbuser/ns-usbuser-_usbuser_power_info_request)<br/>              | The [**USBUSER\_POWER\_INFO\_REQUEST**](/windows/desktop/api/usbuser/ns-usbuser-_usbuser_power_info_request) structure is used in conjunction with the [**IOCTL\_USB\_USER\_REQUEST**](/windows/desktop/api/Usbuser/ni-usbuser-ioctl_usb_user_request) I/O control request to retrieve power policy information concerning the relationship of a specific system state to the power state of the host controller and the root hub.<br/> |
| [**USBUSER\_REQUEST\_HEADER**](/windows/desktop/api/usbuser/ns-usbuser-_usbuser_request_header)<br/>                       | The [**USBUSER\_REQUEST\_HEADER**](/windows/desktop/api/usbuser/ns-usbuser-_usbuser_request_header) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](/windows/desktop/api/Usbuser/ni-usbuser-ioctl_usb_user_request) I/O control request to send a user-mode request to the USB host controller driver.<br/>                                                                                                                  |
| [**WDMUSB\_POWER\_STATE**](/windows/desktop/api/usbuser/ne-usbuser-_wdmusb_power_state)<br/>                               | The [**WDMUSB\_POWER\_STATE**](/windows/desktop/api/usbuser/ne-usbuser-_wdmusb_power_state) enumeration indicates the power state of a host controller or root hub.<br/>                                                                                                                                                                                                                            |
