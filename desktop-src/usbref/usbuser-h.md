# Usbuser.h

This section contains reference topics for the Usbuser.h header.

## In this section



| Topic                                                                                       | Description                                                                                                                                                                                                                                                                                                                                                          |
|---------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| [**IOCTL\_USB\_USER\_REQUEST**](ioctl-usb-user-request.md)<br/>                      | The [**IOCTL\_USB\_USER\_REQUEST**](ioctl-usb-user-request.md) I/O control request is available to both user-mode applications and kernel-mode drivers. <br/>                                                                                                                                                                                                 |
| [**USB\_BANDWIDTH\_INFO**](usb-bandwidth-info.md)<br/>                               | The [**USB\_BANDWIDTH\_INFO**](usb-bandwidth-info.md) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](ioctl-usb-user-request.md) I/O control request to retrieve information about the allocated bandwidth.<br/>                                                                                                                                  |
| [**USB\_BUS\_STATISTICS\_0**](usb-bus-statistics-0.md)<br/>                          | The [**USB\_BUS\_STATISTICS\_0**](usb-bus-statistics-0.md) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](ioctl-usb-user-request.md) I/O control request to retrieve bus statistics.<br/>                                                                                                                                                        |
| [**USB\_CLOSE\_RAW\_DEVICE\_PARAMETERS**](usb-close-raw-device-parameters.md)<br/>   | This structure is not supported. <br/> The [**USB\_CLOSE\_RAW\_DEVICE\_PARAMETERS**](usb-close-raw-device-parameters.md) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](ioctl-usb-user-request.md) I/O control request to close raw access to devices on the bus.<br/>                                                                     |
| [**USB\_CONTROLLER\_INFO\_0**](usb-controller-info-0.md)<br/>                        | The [**USB\_CONTROLLER\_INFO\_0**](usb-controller-info-0.md) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](ioctl-usb-user-request.md) I/O control request to retrieve information about the USB host controller.<br/>                                                                                                                           |
| [**USB\_DRIVER\_VERSION\_PARAMETERS**](usb-driver-version-parameters.md)<br/>        | The [**USB\_DRIVER\_VERSION\_PARAMETERS**](usb-driver-version-parameters.md) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](ioctl-usb-user-request.md) I/O control request to retrieve version information.<br/>                                                                                                                                 |
| [**USB\_PASS\_THRU\_PARAMETERS**](usb-pass-thru-parameters.md)<br/>                  | The [**USB\_PASS\_THRU\_PARAMETERS**](usb-pass-thru-parameters.md) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](ioctl-usb-user-request.md) I/O control request to pass a vendor-specific command to the host controller miniport driver.<br/>                                                                                                  |
| [**USB\_POWER\_INFO**](usb-power-info.md)<br/>                                       | The [**USB\_POWER\_INFO**](usb-power-info.md) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](ioctl-usb-user-request.md) I/O control request to retrieve device power state that the host controller power policy specifies for the indicated system power state.<br/>                                                                            |
| [**USB\_UNICODE\_NAME**](usb-unicode-name.md)<br/>                                   | The [**USB\_UNICODE\_NAME**](usb-unicode-name.md) structure contains a Unicode string that specifies a symbolic link name.<br/>                                                                                                                                                                                                                               |
| [**USB\_USER\_ERROR\_CODE**](usb-user-error-code.md)<br/>                            | The [**USB\_USER\_ERROR\_CODE**](usb-user-error-code.md) enumeration lists the error codes that a USB user-mode request reports when it fails.<br/>                                                                                                                                                                                                           |
| [**USBUSER\_BANDWIDTH\_INFO\_REQUEST**](usbuser-bandwidth-info-request.md)<br/>      | The [**USBUSER\_BANDWIDTH\_INFO\_REQUEST**](usbuser-bandwidth-info-request.md) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](ioctl-usb-user-request.md) I/O control request to retrieve information about the allocated bandwidth.<br/>                                                                                                         |
| [**USBUSER\_BUS\_STATISTICS\_0\_REQUEST**](usbuser-bus-statistics-0-request.md)<br/> | The [**USBUSER\_BUS\_STATISTICS\_0\_REQUEST**](usbuser-bus-statistics-0-request.md) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](ioctl-usb-user-request.md) I/O control request to retrieve bus statistics.<br/>                                                                                                                               |
| [**USBUSER\_CONTROLLER\_UNICODE\_NAME**](usbuser-controller-unicode-name.md)<br/>    | The [**USBUSER\_CONTROLLER\_UNICODE\_NAME**](usbuser-controller-unicode-name.md) structure is used in conjunction with the [**IOCTL\_USB\_USER\_REQUEST**](ioctl-usb-user-request.md) I/O control request to retrieve the USB host controller driverkey name.<br/>                                                                                           |
| [**USBUSER\_GET\_DRIVER\_VERSION**](usbuser-get-driver-version.md)<br/>              | The [**USBUSER\_GET\_DRIVER\_VERSION**](usbuser-get-driver-version.md) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](ioctl-usb-user-request.md) I/O control request to read driver and interface version information.<br/>                                                                                                                      |
| [**USBUSER\_PASS\_THRU\_REQUEST**](usbuser-pass-thru-request.md)<br/>                | The [**USBUSER\_PASS\_THRU\_REQUEST**](usbuser-pass-thru-request.md) structure is used in conjunction with the [**IOCTL\_USB\_USER\_REQUEST**](ioctl-usb-user-request.md) I/O control request to send a vendor-specific command to the host controller miniport driver.<br/>                                                                                 |
| [**USBUSER\_POWER\_INFO\_REQUEST**](usbuser-power-info-request.md)<br/>              | The [**USBUSER\_POWER\_INFO\_REQUEST**](usbuser-power-info-request.md) structure is used in conjunction with the [**IOCTL\_USB\_USER\_REQUEST**](ioctl-usb-user-request.md) I/O control request to retrieve power policy information concerning the relationship of a specific system state to the power state of the host controller and the root hub.<br/> |
| [**USBUSER\_REQUEST\_HEADER**](usbuser-request-header.md)<br/>                       | The [**USBUSER\_REQUEST\_HEADER**](usbuser-request-header.md) structure is used with the [**IOCTL\_USB\_USER\_REQUEST**](ioctl-usb-user-request.md) I/O control request to send a user-mode request to the USB host controller driver.<br/>                                                                                                                  |
| [**WDMUSB\_POWER\_STATE**](wdmusb-power-state.md)<br/>                               | The [**WDMUSB\_POWER\_STATE**](wdmusb-power-state.md) enumeration indicates the power state of a host controller or root hub.<br/>                                                                                                                                                                                                                            |
