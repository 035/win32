# H

<dl> <dt>

<span id="_netmon_handoff_set_gly"></span><span id="_NETMON_HANDOFF_SET_GLY"></span>**handoff set**
</dt> <dd>

A protocol set that follows a protocol. A parser uses the handoff set of a protocol if the parser can detect the next protocol from the data in the capture. See also [*follow set*](f.md#-netmon-follow-set-gly)

</dd> <dt>

<span id="_netmon_handoff_table_gly"></span><span id="_NETMON_HANDOFF_TABLE_GLY"></span>**handoff table**
</dt> <dd>

A table that contains the parser handoff set. You can use a handoff table to access the handoff set information — without accessing the parser-defined INI file. The handoff set information is stored in the INI file of each parser. See also *handoff set*

</dd> <dt>

<span id="_netmon_hex_pane_gly"></span><span id="_NETMON_HEX_PANE_GLY"></span>**hex pane**
</dt> <dd>

The pane in the Network Monitor frame viewer window that shows the hexadecimal bytes transmitted in frames that are included in a capture. By default, the hex pane appears at the bottom of the window.

</dd> </dl>
