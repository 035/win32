# IPAM Server WMI Provider Reference

This section contains the classes of the IPAM Server WMI provider.

## In this section

<dl> <dt>

[**CIM\_AdminDomain**](cim-admindomain.md)
</dt> <dd>

**CIM\_AdminDomain** represents a group of systems that are administered as a single entity based on a common user, user group, or policy.

</dd> <dt>

[**CIM\_Collection**](cim-collection.md)
</dt> <dd>

[**CIM\_Collection**](cim-collection.md) is an abstract base class that represents a collection of managed elements.

</dd> <dt>

[**CIM\_CollectionOfMSEs**](cim-collectionofmses.md)
</dt> <dd>

Represents a stateless collection of managed system elements, and their settings.

</dd> <dt>

[**CIM\_Component**](cim-component.md)
</dt> <dd>

[**CIM\_Component**](cim-component.md) is a generic association used to establish the parts of a relationship between Managed System Elements.

</dd> <dt>

[**CIM\_ConnectivityCollection**](cim-connectivitycollection.md)
</dt> <dd>

Represents a collection of protocol endpoints that are of the same class type and can communicate with each other. In addition, **CIM\_ConnectivityCollection** can contain systems, users, and managed elements that are related to the protocol endpoints.

</dd> <dt>

[**CIM\_ConcreteJob**](cim-concretejob.md)
</dt> <dd>

A instantiable unit of work to run, such as a batch or a print job.

</dd> <dt>

[**CIM\_DatabaseParameter**](cim-databaseparameter.md)
</dt> <dd>

The **CIM\_DatabaseParameter** class is an abstract base class used to define database and service configurations that are common across database models and vendor implementations.

</dd> <dt>

[**CIM\_Error**](cim-error.md)
</dt> <dd>

The **CIM\_Error** class contains information about the failure of a CIM operation.

</dd> <dt>

[**CIM\_EnabledLogicalElement**](cim-enabledlogicalelement.md)
</dt> <dd>

Represents a logical element that can be enabled and disabled.

</dd> <dt>

[**CIM\_IPConnectivitySubnet**](cim-ipconnectivitysubnet.md)
</dt> <dd>

Represents a subnet that contains IP protocol endpoints.

</dd> <dt>

[**CIM\_IPProtocolEndpoint**](cim-ipprotocolendpoint.md)
</dt> <dd>

**CIM\_IPProtocolEndpoint** is a protocol endpoint that is dedicated to running Internet Protocol (IP).

</dd> <dt>

[**CIM\_Job**](cim-job.md)
</dt> <dd>

A logical element that represents a unit of work to execute, such as a script or a print job. A job is distinct from a process because a job can be scheduled or queued, and its execution is not limited to a single system.

</dd> <dt>

[**CIM\_LogEntry**](cim-logentry.md)
</dt> <dd>

Represents a log entry.

</dd> <dt>

[**CIM\_LogicalElement**](cim-logicalelement.md)
</dt> <dd>

[**CIM\_LogicalElement**](cim-logicalelement.md) is a base class for all the components of a System that represent abstract system components, such as Files, Processes, or LogicalDevices.

</dd> <dt>

[**CIM\_ManagedElement**](cim-managedelement.md)
</dt> <dd>

The **CIM\_ManagedElement** class is an abstract class that provides a common superclass (or top of the inheritance tree) for the non-association classes in the CIM Schema.

</dd> <dt>

[**CIM\_ManagedSystemElement**](cim-managedsystemelement.md)
</dt> <dd>

CIM\_ManagedSystemElement is the base class for the System Element hierarchy. Any distinguishable component of a System is a candidate for inclusion in this class. Examples of system components include: - software components such as application servers, databases, and applications - operating system components such as files, processes, and threads - device components such as disk drives, controllers, processors, and printers - physical components such as chips and cards.

</dd> <dt>

[**CIM\_Product**](cim-product.md)
</dt> <dd>

The **CIM\_Product** class represents a set of goods or services produced by a supplier as a single unit for a user. A product can include physical elements, software, or services.

</dd> <dt>

[**CIM\_ProtocolEndpoint**](cim-protocolendpoint.md)
</dt> <dd>

**CIM\_ProtocolEndpoint** is a communication point that sends and receives data between system or computer interfaces and logical networks.

</dd> <dt>

[**CIM\_RangeOfIPAddresses**](cim-rangeofipaddresses.md)
</dt> <dd>

Represents an IP address range defined by a start and end address.

</dd> <dt>

[**CIM\_RecordForLog**](cim-recordforlog.md)
</dt> <dd>

Represents a record for a log.

</dd> <dt>

[**CIM\_ScopedSettingData**](cim-scopedsettingdata.md)
</dt> <dd>

The **CIM\_ScopedSettingData** class is an abstract classed used to define settings data for a managed element.

</dd> <dt>

[**CIM\_ServiceAccessPoint**](cim-serviceaccesspoint.md)
</dt> <dd>

[**CIM\_ServiceAccessPoint**](cim-serviceaccesspoint.md) represents the ability to utilize or invoke a Service. Access points represent that a Service is made available for other entities to use.

</dd> <dt>

[**CIM\_Setting**](cim-setting.md)
</dt> <dd>

The **CIM\_Setting** class represents a configuration or an operational parameter for one or more managed system elements.

</dd> <dt>

[**CIM\_SettingData**](cim-settingdata.md)
</dt> <dd>

The **CIM\_SettingData** class represents configuration and operational parameters for **CIM\_ManagedElement** instances.

</dd> <dt>

[**CIM\_SoftwareFeature**](cim-softwarefeature.md)
</dt> <dd>

Represents a feature of a deployed software installation.

</dd> <dt>

[**CIM\_System**](cim-system.md)
</dt> <dd>

CIM\_System represents a set of components that function as a single high-level entity.

</dd> <dt>

[**CIM\_SystemSpecificCollection**](cim-systemspecificcollection.md)
</dt> <dd>

Represents a collection of objects that are contained within a system.
