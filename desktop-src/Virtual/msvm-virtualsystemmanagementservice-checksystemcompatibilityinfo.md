# CheckSystemCompatibilityInfo method of the Msvm\_VirtualSystemManagementService class

Verifies whether this computer system is able to host the specified virtual machine (VM).

## Syntax


```mof
uint32 CheckSystemCompatibilityInfo(
  [in]  uint8  CompatibilityInfo[],
  [out] string Reasons[]
);
```



## Parameters

<dl> <dt>

*CompatibilityInfo* \[in\]
</dt> <dd>

Type: **uint8\[\]**

An opaque blob generated by the [**GetSystemCompatibilityInfo**](msvm-virtualsystemmanagementservice-getsystemcompatibilityinfo.md) method that represents either one VM or all VMs hosted by a computer system.

</dd> <dt>

*Reasons* \[out\]
</dt> <dd>

Type: **string\[\]**

An array of strings containing [**Msvm\_Error**](msvm-error.md) instances in CIM-XML format representing warnings or error information. These strings can be passed to the [**FormatError**](formaterror-msvm-virtualsystemmanagementservice.md) method to generate error messages.

</dd> </dl>

## Return value

Type: **uint32**

This method returns one of the following values. A return of 0 (Completed with No Error) indicates the systems are compatible. A return of 32784 (Not compatible) indicates that the systems are not compatible. Any other return value indicates an error checking the compatibility information. If **CheckSystemCompatibilityInfo** returns either 0 or 32784, the *Reasons* parameter may contain more detailed warning or error information.

## Remarks

The [**GetSystemCompatibilityInfo**](msvm-virtualsystemmanagementservice-getsystemcompatibilityinfo.md) method is used in conjunction with **CheckSystemCompatibilityInfo** to determine whether a quick or live migration of a VM to another hosting computer system is possible without first trying the migration. For more details, see the Remarks section of [**GetSystemCompatibilityInfo**](msvm-virtualsystemmanagementservice-getsystemcompatibilityinfo.md).

Access to the [**Msvm\_VirtualSystemManagementService**](msvm-virtualsystemmanagementservice.md) class might be restricted by UAC Filtering. For more information, see [User Account Control and WMI](https://msdn.microsoft.com/library/aa826699).
