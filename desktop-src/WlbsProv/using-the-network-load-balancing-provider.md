# Using the Network Load Balancing Provider

The Network Load Balancing (NLB) [*Provider*](https://msdn.microsoft.com/library/aa371764#mscs-n---z-9-gly) defines [*WMI classes*](https://msdn.microsoft.com/library/aa371764#mscs-n---z-22-gly) that allow your application to control NLB [*clusters*](https://msdn.microsoft.com/library/aa367183#mscs-a---e-5-gly), [*nodes*](https://msdn.microsoft.com/library/aa371764#mscs-n---z-5-gly), and [*port rules*](https://msdn.microsoft.com/library/aa371764#mscs-n---z-7-gly).

For information on how to use the Network Load Balancing provider, see the following sections:

-   [Enumerating Remote Nodes](enumerating-remote-nodes.md)
-   [Making Direct Connections](making-direct-connections.md)
-   [Monitoring Application Level Health](https://msdn.microsoft.com/library/cc307934)
-   [Obtaining Class Instances](obtaining-class-instances.md)

For example code demonstrating how to use the provider, see the [scripting example](scripting-example.md).

For general information about how to use providers, see [Windows Management Instrumentation](https://msdn.microsoft.com/library/aa394582)(WMI).
