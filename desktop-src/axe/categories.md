# Categories element

Categories that enable the UX to handle multiple items.

## Usage

``` syntax
<Categories>
  child elements
</Categories>
```

## Attributes

There are no attributes.

## Child elements



| Element                                 | Description                                                                  |
|-----------------------------------------|------------------------------------------------------------------------------|
| [**Category**](category.md)<br/> | Describes a category to which the assessment belongs.<br/> <br/> |



### Child element sequence

``` syntax
Category
```

## Parent elements



| Element                                       | Description                                                   |
|-----------------------------------------------|---------------------------------------------------------------|
| [**Description**](description.md)<br/> | Provides a human-readable description.<br/> <br/> |



## Element information



|              |     |
|--------------|-----|
| Can be empty | No  |
