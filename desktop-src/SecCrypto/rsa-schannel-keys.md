# RSA/Schannel Keys

## Generating and Retrieving RSA/Schannel Keys

[*RSA*](security.r_gly#-security-rsa-gly)/[*Schannel*](security.s_gly#-security-schannel-gly) keys can be generated by calling the [**CryptGenKey**](cryptgenkey.md) function. The call to **CryptGenKey** requires an AT\_KEYEXCHANGE algorithm identifier passed in the *Algid* parameter.

**To generate an RSA/Schannel public/private key pair**

1.  Call the [**CryptAcquireContext**](cryptacquirecontext.md) function to get a handle to the [Microsoft RSA/Schannel Cryptographic Provider](microsoft-rsa-schannel-cryptographic-provider.md).
2.  Call the [**CryptGenKey**](cryptgenkey.md) function to generate the keys. AT\_KEYEXCHANGE must be passed in for the *Algid* parameter, and the upper 16 bits of the *dwFlags* parameter must be set to the desired key size (512 bits). An [**HCRYPTKEY**](hcryptkey.md) structure handle is returned in the *hKey* parameter.

**To retrieve a pointer to previously generated RSA/Schannel user keys**

1.  Call the [**CryptAcquireContext**](cryptacquirecontext.md) function to get a handle to the [Microsoft RSA/Schannel Cryptographic Provider](microsoft-rsa-schannel-cryptographic-provider.md).
2.  Call the [**CryptGetUserKey**](cryptgetuserkey.md) function, with the *dwKeySpec* parameter set to AT\_KEYEXCHANGE.

## Exporting RSA/Schannel Keys

[*Master keys*](security.m_gly#-security-master-key-gly) can be exported into simple key BLOB structures. This should be implemented the same way as the export of normal [*RC4*](security.r_gly#-security-rc4-gly) or [*Data Encryption Standard*](security.d_gly#-security-data-encryption-standard-gly) (DES) [*bulk encryption keys*](security.b_gly#-security-bulk-encryption-key-gly), as described in [Simple Key BLOB](seccngprov.simple_key_blobs). The **aiKeyAlg** member of the [**PUBLICKEYSTRUC**](publickeystruc.md) structure is set to the algorithm identifier of the master key (CALG\_PCT1\_MASTER, CALG\_SSL2\_MASTER, CALG\_SSL3\_MASTER, or CALG\_TLS1\_MASTER).

If the [**CPExportKey**](seccngprov.cpexportkey) function is exporting an SSL2 master key and the CRYPT\_SSL2\_FALLBACK flag is set, then, to help prevent version rollback attacks, set the first eight bytes of the encryption block [*padding*](security.p_gly#-security-padding-gly) to 0x03 rather than to random data.

If the **CRYPT\_DESTROYKEY** constant is specified in the *dwFlags* parameter of the [**CPExportKey**](seccngprov.cpexportkey) function, the CSP destroys the key or key handle after exporting the key. This flag is intended for use only with [*opaque BLOBs*](security.o_gly#-security-opaque-blob-gly).
