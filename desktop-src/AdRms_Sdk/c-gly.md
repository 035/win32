# C

\[The AD RMS SDK leveraging functionality exposed by the client in Msdrm.dll is available for use in Windows Server 2008, Windows Vista, Windows Server 2008 R2, Windows 7, Windows Server 2012, and Windows 8. It may be altered or unavailable in subsequent versions. Instead, use [Active Directory Rights Management Services SDK 2.1](https://msdn.microsoft.com/library/hh535290), which leverages functionality exposed by the client in Msipc.dll.\]

<dl> <dt>

<span id="_rm_certificate_gly"></span><span id="_RM_CERTIFICATE_GLY"></span>**certificate**
</dt> <dd>

A signed statement that associates an identity to a pair of electronic keys that can be used to encrypt and sign digital information.

</dd> <dt>

<span id="_rm_client_enrollment_gly"></span><span id="_RM_CLIENT_ENROLLMENT_GLY"></span>**client enrollment**
</dt> <dd>

The process of creating the client licensor certificate, which enables the user's computer or device to create issuance licenses that will be honored by a licensing server.

</dd> <dt>

<span id="_rm_client_licensor_certificate_gly"></span><span id="_RM_CLIENT_LICENSOR_CERTIFICATE_GLY"></span>**client licensor certificate**
</dt> <dd>

The certificate created by an AD RMS server and placed on AD RMS client computers, which enables users to publish protected content offline, without being connected to the AD RMS-enabled network.

</dd> <dt>

<span id="_rm_condition_gly"></span><span id="_RM_CONDITION_GLY"></span>**condition**
</dt> <dd>

A set of specified constraints and parameters that are part of the rights group bundled into a issuance license. These are enforced at the time of consumption.

</dd> <dt>

<span id="_rm_content_key_gly"></span><span id="_RM_CONTENT_KEY_GLY"></span>**content key**
</dt> <dd>

The key used to both encrypt and decrypt protected content during publishing and consumption.

</dd> <dt>

<span id="_rm_content_owner_gly"></span><span id="_RM_CONTENT_OWNER_GLY"></span>**content owner**
</dt> <dd>

The person or organization that controls access to protected content.

</dd> </dl>
