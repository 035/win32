# G

<dl> <dt>

<span id="_wolf_generic_application_resource_gly"></span><span id="_WOLF_GENERIC_APPLICATION_RESOURCE_GLY"></span>**Generic Application resource**
</dt> <dd>

An application being managed as a cluster [*resource*](r-gly.md#-wolf-resource-gly). The Cluster service defines the Generic Application [*resource type*](r-gly.md#-wolf-resource-type-gly) for managing Generic Application resources.

</dd> <dt>

<span id="_wolf_generic_service_resource_gly"></span><span id="_WOLF_GENERIC_SERVICE_RESOURCE_GLY"></span>**Generic Service resource**
</dt> <dd>

A service being managed as a cluster [*resource*](r-gly.md#-wolf-resource-gly). The Cluster service defines the Generic Service [*resource type*](r-gly.md#-wolf-resource-type-gly) for managing Generic Service resources.

</dd> <dt>

<span id="_wolf_global_update_manager_gly"></span><span id="_WOLF_GLOBAL_UPDATE_MANAGER_GLY"></span>**Global Update Manager**
</dt> <dd>

Software component in the [*Cluster service*](c-gly.md#-wolf-cluster-service-gly) that provides update services to the [*nodes*](n-gly.md#-wolf-node-gly) in a [*cluster*](c-gly.md#-wolf-cluster-gly).

</dd> <dt>

<span id="_wolf_group_gly"></span><span id="_WOLF_GROUP_GLY"></span>**group**
</dt> <dd>

A collection of [*resources*](r-gly.md#-wolf-resource-gly) managed as a single [*cluster object*](c-gly.md#-wolf-cluster-object-gly). Typically a group contains all of the resources needed to run a specific application or service. [*Failover*](f-gly.md#-wolf-failover-gly) and [*failback*](f-gly.md#-wolf-failback-gly) always act on groups.

</dd> </dl>
