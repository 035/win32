# D

<dl> <dt>

<span id="_wolf_database_manager_gly"></span><span id="_WOLF_DATABASE_MANAGER_GLY"></span>**Database Manager**
</dt> <dd>

Software component in the [*Cluster service*](https://www.bing.com/search?q=*Cluster service*) that controls access to the [*cluster database*](https://www.bing.com/search?q=*cluster database*).

</dd> <dt>

<span id="_wolf_data_value_gly"></span><span id="_WOLF_DATA_VALUE_GLY"></span>**data value**
</dt> <dd>

A single value associated with a property value. A [*cluster object*](https://www.bing.com/search?q=*cluster object*) property consists of one or more property values, each consisting of one or more data values. For example, if a property consists of multiple **DWORD** arrays, each array is considered a property value, and each array element is considered a data value.

</dd> <dt>

<span id="_wolf_dependency_gly"></span><span id="_WOLF_DEPENDENCY_GLY"></span>**dependency**
</dt> <dd>

A [*resource*](https://www.bing.com/search?q=*resource*) on which another resource depends. If resource A depends on resource B, then B is a dependency of A.

</dd> <dt>

<span id="_wolf_dependency_relationship_gly"></span><span id="_WOLF_DEPENDENCY_RELATIONSHIP_GLY"></span>**dependency relationship**
</dt> <dd>

Relationship between two [*resources*](https://www.bing.com/search?q=*resources*) where one resource depends on another resource to operate. Resources in a dependency relationship must be in the same [*group*](https://www.bing.com/search?q=*group*). Dependent resources are taken [*offline*](https://www.bing.com/search?q=*offline*) before, and brought [*online*](https://www.bing.com/search?q=*online*) after, their [*dependencies*](#-wolf-dependency-gly).

</dd> <dt>

<span id="_wolf_dependent_gly"></span><span id="_WOLF_DEPENDENT_GLY"></span>**dependent**
</dt> <dd>

A [*resource*](https://www.bing.com/search?q=*resource*) that depends on one or more resources in order to operate. If resource A depends on resource B, then resource A is dependent.

</dd> <dt>

<span id="_wolf_disk_mirroring_gly"></span><span id="_WOLF_DISK_MIRRORING_GLY"></span>**disk mirroring**
</dt> <dd>

Fault tolerant technique that writes data simultaneously to two disks using the same disk controller. The disks operate in tandem, constantly storing and updating the same files. Mirroring alone does not ensure data protection. If both disks fail at the same time, you still lose data.

</dd> <dt>

<span id="_wolf_disk_striping_gly"></span><span id="_WOLF_DISK_STRIPING_GLY"></span>**disk striping**
</dt> <dd>

Spreading data over multiple disk drives. Data is interleaved by byte or sector blocks across the drives.

</dd> <dt>

<span id="_wolf_domain_name_system_gly"></span><span id="_WOLF_DOMAIN_NAME_SYSTEM_GLY"></span>**Domain Name System (DNS)**
</dt> <dd>

Service that maps [*node*](https://www.bing.com/search?q=*node*) names to IP addresses.

</dd> <dt>

<span id="_wolf_dword_aligned_gly"></span><span id="_WOLF_DWORD_ALIGNED_GLY"></span>**DWORD-aligned**
</dt> <dd>

The alignment or spacing of data in a buffer. Data is **DWORD**-aligned if the byte size of each data value is an even multiple of a **DWORD** (4 bytes).

</dd> </dl>
