<dl> <dt>

<span id="_wolf_quorum_capable_resource_gly"></span><span id="_WOLF_QUORUM_CAPABLE_RESOURCE_GLY"></span>**quorum-capable resource**
</dt> <dd>

A [*storage class resource*](https://www.bing.com/search?q=*storage class resource*) that is an instance of a [*resource type*](https://www.bing.com/search?q=*resource type*) implemented to support [*quorum resources*](#-wolf-quorum-resource-gly). By default, only [*Physical Disk resources*](https://www.bing.com/search?q=*Physical Disk resources*) are quorum-capable, but developers can create resource types that support other kinds of storage class resources as quorum-capable resources.

</dd> <dt>

<span id="_wolf_quorum_log_gly"></span><span id="_WOLF_QUORUM_LOG_GLY"></span>**quorum log**
</dt> <dd>

See [*recovery log*](https://www.bing.com/search?q=*recovery log*).

</dd> <dt>

<span id="_wolf_quorum_resource_gly"></span><span id="_WOLF_QUORUM_RESOURCE_GLY"></span>**quorum resource**
</dt> <dd>

The [*quorum-capable resource*](#-wolf-quorum-capable-resource-gly) selected to maintain essential cluster data. The quorum resource stores a synchronized version of the [*cluster database*](https://www.bing.com/search?q=*cluster database*) as well as critical recovery information in a [*recovery log*](https://www.bing.com/search?q=*recovery log*). The quorum resource guarantees that all [*nodes*](https://www.bing.com/search?q=*nodes*) have access to the most recent database changes.

</dd> </dl>
